# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**landau.py** is a Python library for calculating thermodynamic equilibria and plotting phase diagrams in the (semi-)grand ensemble. It enables computational thermodynamics research, particularly for alloy systems.

## Working Style (read this first)

These are project-specific preferences distilled from past PR/issue feedback. Some restate harness defaults but are repeated here because they have been violated often enough to matter.

### Tone in PRs, commits, and comments
- Match a terse, imperative tone. No "Happy to...", no apologies, no enthusiasm filler. (Restates the harness default — keep it in mind anyway.)
- PR bodies are plain technical reports. No **bold** "headline numbers", no "Conclusion" sections, no `[Fix this →]` action links, no marketing claims ("dramatic speedup", "significantly improves").
- Do not invoke the maintainer's name or authority in PR bodies or comments (PR #94: "don't use my name in vain").
- Keep the PR description (the OP) in sync with the diff. When review feedback or later commits invalidate something in the body, edit the body — never leave a stale claim standing (e.g. a dropped export still described as exported, an outdated test count).

### Comments and docstrings
- Reflect only the current state of the code. Historical motivation is acceptable (why a threshold exists, why an approach was chosen), but never narrate the intermediate steps, rejected alternatives, or "old 10% vs new 1%" comparisons that arose during a PR. Those belong in the PR description, not in the source.
- Do not reference thresholds, approaches, or behaviour that no longer exist in the code.

### Evidence over claims
- Cite the specific commit hash, file path, test count, or command output backing any claim. If a number appears in the PR body, the script that produced it goes in `benchmarks/` in the same PR (PR #94).
- For physics/numerics claims, prefer saying nothing over saying something unverified (PR #112: "Rather say nothing than something wrong").
- If asked to verify, actually verify — read the docs, run the code. Do not re-assert. (PR #113: "I'm not entirely sure I believe you. Read the docs and get back to me.")

### Scope and abstraction
- Prefer the minimal change. "I'd rather merge less features now and extend later than break API" (PR #68).
- Before writing a new helper, check the codebase and existing deps (numpy, scipy, shapely, matplotlib.testing, pyiron_snippets) for one that already does the job. Reuse over reimplement (PR #115).
- One purpose per PR. Notebooks, benchmarks, refactors, and unrelated fixes go in their own PRs (PR #68, #82, #85). Notebooks are committed with executed outputs only.
- When a PR grows a second concept, propose splitting it before pushing more.
- Speculative or "might be useful" work gets parked, not landed (PR #129 closed, #130 parked).

### Test quality bar
(Expands on the Testing section below.)
- Tests must assert tight conditions a degenerate/constant solution would fail. Loose `atol=0.05` "it ran" tests get rejected (PR #80, #101).
- Use Hypothesis for round-trip recovery (random coeffs in → `fit()` → coeffs out) where applicable (PR #82).
- Share a single `atol` constant across related tests rather than scattering magic numbers (PR #82).
- Use the module-wide pytest skip marker for optional-dep tests (PR #111).

### Debugging numerics visually
- When diagnosing a fit/solver/physics issue, generate a plot of the relevant landscape (loss vs parameter, etc.), attach it inline to the PR comment, and add a walkthrough section to a notebook so the maintainer can step through interactively (PR #82, #115).

### Naming and layout
- Module names: no underscores. `asewrapper`, not `ase_wrapper` (PR #68).
- Promote single-file modules to subpackages once they grow (as was done for `interpolate/`).

### Git workflow
- Merge commits are disabled on GitHub; never bring a branch up to date with `git merge`. Rebase onto `origin/main` instead (`git reset` off any stray merge, then `git rebase origin/main`) and force-push, so history stays linear.

### Do not commit
- `.hypothesis/`, `_version.py`, stray top-level scripts, duplicate exploratory files (PR #71, #94). Check for duplicates of what you are about to add before pushing.

### Pandas 2/3 compatibility (hard constraint)
- All `groupby().apply()` calls must pass `include_groups=False`.
- Code must work on both pandas 2 and 3. Do not drop pandas 2 (PR #93, #113).
- The fix in commit `945f850` replaced deprecated `include_groups=True`; regressions are pinned in `tests/regression/test_single_group_apply.py` and `test_cluster_phase_pandas3.py`.

## Architecture

### Module layout

**`landau/phases/`** — phase definitions (subpackage; split from `phases.py` per PR #111). The user-facing concrete classes (`LinePhase`, `TemperatureDep[ae]ndantLinePhase`, `IdealSolution`, `RegularSolution`, `InterpolatingPhase`, `SlowInterpolatingPhase`, `FastInterpolatingPhase`, `AsePhase`) are re-exported from `landau/__init__.py`; the `Phase` / `AbstractLinePhase` bases stay under `landau.phases`, and the point-defect family now lives in `landau.phases.pointdefects` (pre-split names kept importable from `landau.phases` for back-compat — see below).
- `phases/__init__.py` — `Phase` (ABC), `AbstractLinePhase`, `LinePhase`, `TemperatureDependentLinePhase` (alias `TemperatureDepandantLinePhase` kept for back-compat), `IdealSolution`, `RegularSolution`, `InterpolatingPhase`, `SlowInterpolatingPhase`, `FastInterpolatingPhase`. The `_scalarize(x)` helper (PR #208; now defined in `landau/interpolate/basic.py` and imported back here — moved down to the lower `interpolate` layer so `SplineFit` can share it without an import cycle) is the single site that collapses 0-d numpy results back to Python scalars so the scalar-in/scalar-out contract holds across `AbstractLinePhase.concentration`, `IdealSolution.semigrand_potential`, `RegularSolution.semigrand_potential`, and `(Slow|Fast)InterpolatingPhase._find_phi_c` — sites that previously hand-rolled the same `if ndim == 0` / `if shape == ()` / `isinstance(x, np.ndarray)` check and had drifted. The point defect classes and `AsePhase` are defined in sibling modules and imported back at the end of `__init__.py` (after `Phase`/`AbstractLinePhase`/`kB` exist, so the `from . import …` in those modules resolves). Only the **pre-split** point-defect API is re-exported here for back-compat: `AbstractPointDefect` (ABC) as a plain alias (stays subclassable), the concrete `ConstantPointDefect`/`PointDefectSublattice`/`PointDefectedPhase` behind `@deprecate(..., version="2.0")` factory shims (warn, point at `landau.phases.pointdefects`) — these and the matching back-compat tests in `tests/unit/test_phases.py` are to be removed at 2.0. The classes added with the split (`AbstractPointDefectSublattice`, `LowTemperatureExpansionSublattice`) are **not** re-exported and have no entry in `phases/__init__.py:__all__`; import them from `landau.phases.pointdefects`.
- `SlowInterpolatingPhase` / `FastInterpolatingPhase` — `SlowInterpolatingPhase` is the per-scalar `scipy.optimize.brute` reference oracle; `FastInterpolatingPhase` (subclass; PR #281) is a vectorized exact replacement and the default user-facing choice (`brute`'s 20-point grid can miss a deep narrow well — PR #281 benchmarks show 25-33 misses out of 24k on a random-phase sweep). For one `T` it builds `f(c) = fe(c) - T*S(c)` once on a 201-point grid (`_n_grid` class var), takes one vectorized `argmin` over `dmu` to locate the global basin (handles miscibility gaps), then polishes with 6 Newton steps (`_n_newton`) in the logit variable `u = log(c/(1-c))` — the ideal-mixing entropy contributes `-T*S'(c) = kB*T*u` which is *linear* in `u`, so the polish is uniformly well-conditioned from the dilute to the saturated limit (where a plain `c`-space Newton step is stiff). The polish is confined to the grid cell around the basin, and the lowest of `{Newton, cl, cr}` is kept so a boundary minimum is recovered exactly. Uses `fe.deriv()` (analytic for `PolyFit`/`RedlichKister`) so the located `c` is the exact stationary point and `c = -∂φ/∂Δμ` holds to machine precision. A bytes-keyed `_find_phi_c_cached` (`@lru_cache(maxsize=512)`, keyed on `(shape, tobytes())` since arrays are unhashable) lets `semigrand_potential` and `concentration` reuse one solve — `calc_phase_diagram` calls them separately with the same `(T, dmu)`. Two orders of magnitude faster than `SlowInterpolatingPhase` on representative phases (`benchmarks/bench_fast_interpolating_phase.py`). Both subclasses share the `concentration_range` / `maximum_extrapolation` XOR validation in `SlowInterpolatingPhase.__post_init__` (PR #278): explicit `concentration_range` is honoured (was previously silently overwritten by `maximum_extrapolation=0`'s derived range); passing both non-default raises `ValueError`.
- `phases/pointdefects.py` — point-defect phases (split out of `__init__.py`): `AbstractPointDefect` (ABC), `ConstantPointDefect`, `AbstractPointDefectSublattice` (ABC; holds the `name`/`sublattice`/`sublattice_fraction`/`defects` fields, the shared `_get_zes` and the `semigrand_potential_contribution`/`concentration_contribution` combiners, with two abstract hooks `_site_log_partition(s)` → per-site potential in units of `-kB*T` and `_site_normalization(s)` → site-fraction denominator `x_i = z_i / D`; the two are coupled by `c = -∂φ/∂Δμ` ⇒ `D = 1/L'`), `PointDefectSublattice` (exact: `L = ln(1 + Σ_i z_i)`, `D = 1 + Σz_i`, i.e. site competition), `LowTemperatureExpansionSublattice` (dilute leading term: `L = Σz_i`, `D = 1`, so `x_i = z_i` are independent/unnormalised; converges to the exact model when `z_i ≪ 1`) — the two concrete models are **siblings** under the ABC, not a subclass chain. `PointDefectedPhase` (host `AbstractLinePhase` + list of `AbstractPointDefectSublattice`, summing their contributions). The unnormalised LTE site fractions would otherwise drive the total `c` out of `[0, 1]` and `φ → -∞` outside the dilute regime, so `PointDefectedPhase` **clamps `c` to `[0, 1]` and continues `φ` as the tangent line past each saturation point** (`_clamp_fixed_T`): where `c` would exceed 1 it becomes a line phase at `c = 1` (`φ` slope `-1`), where it would fall below 0 a line phase at `c = 0` (`φ` slope `0`). This keeps `c = -∂φ/∂Δμ` consistent *and* bounds `φ` so an out-of-range LTE phase can't spuriously win `argmin φ`. The saturation chemical potentials are precomputed per `T` via `_saturation_window` (bracketed within `_safe_dmu_bound`, below the `exp` overflow threshold), so the raw expression is only evaluated for `dmu` inside the window. A no-op for the bounded exact model (`c` never leaves `(0,1)`, both crossings `±inf`). Imported via `from . import Phase, AbstractLinePhase, kB, _scalarize`.
- `phases/asewrapper.py` — `AsePhase(AbstractLinePhase)` wrapping `ase.thermochemistry.ThermoChem`. Compares/hashes by `pickle.dumps(thermochem)` so two instances from equivalent inputs are equal. Falls back from `get_helmholtz_energy` to `get_gibbs_energy` (1 atm default). `atoms_per_formula` divides ASE's energy so the result is per atom. Splitting this further is open (#137).

**`landau/interpolate/`** — interpolation strategies.
- `basic.py` — `Interpolator` (ABC, the *factory*: `fit(x, y) → Interpolation`), `Interpolation` (ABC, the *fitted curve*: callable `__call__(x)` plus `deriv() → Interpolation` returning `f'` as another interpolation — PR #281 added the ABC so `FastInterpolatingPhase` can take an analytic first derivative instead of finite differences). The default `Interpolation.deriv()` returns a `NumericalDerivative` (vectorised scale-aware central difference at relative step `1e-6`); closed forms override it — `PolyFit.fit` returns a `PolynomialInterpolation` (wraps `np.poly1d`; `coefficients` exposed in ascending power order, decoupling callers from poly1d's descending order), `SGTE.fit` returns an `SGTEInterpolation` with the analytic `dG/dT`, `RedlichKister.fit` returns a `RedlichKisterInterpolation` whose `deriv()` reuses `_eval_mix_derivative`. Stitched / softplus / Whitney still return closure-based interpolations wrapped in `_CallableInterpolation`, which inherits the numerical default. Splits along the `Interpolator` axis: `TemperatureInterpolator` / `ConcentrationInterpolator` (type-hint-only subclasses; interface identical). Concrete: `PolyFit` (TemperatureInterpolator + ConcentrationInterpolator), `SplineFit` (`ConcentrationInterpolator` wrapping `scipy.interpolate.UnivariateSpline`; `smoothing=0` strict interpolation, `>0` smoothing, `None` scipy default; `degree` clamped to `len(x)-1`; extrapolation extends the edge segment per scipy's default `ext=0`; PR #274), `SGTE`, `RedlichKister` (+ `RedlichKisterInterpolation` helper), `StitchedFit`, `G_calphad` standalone fn. Also home to the `_scalarize(x)` 0-d→Python-scalar helper (relocated here from `landau.phases` so both layers share one copy; `landau.phases` re-imports it). `S(c)` (ideal-mixing entropy `kB*(c*ln(c) + (1-c)*ln(1-c))`-shaped) lives in `landau/phases/__init__.py` and is used by every solution-phase free-energy computation; direct unit tests in `tests/unit/test_phases.py` (PR #283, closed #269).
- `softplus.py` — `SoftplusFit` smooth-step (1-D), plus `SoftplusSurface2DInterpolator` (a `SurfaceInterpolator` for `Surface2DInterpolatingPhase`) + its `SoftplusFittedSurface`. The 2-D variant fits `H(T,c) = off(T) + Σ_i a_i(T)·softplus(b_i(T)·(cn+c_i(T)))` in one analytic-Jacobian least-squares: c is the principal variable, each coeff a polynomial in normalised T (`b_order` defaults higher — slope is the strongest T-mover). Amplitudes go through a non-negative link `a_i(T)=softplus(αᵢ(T))` (`_softplus_inv` is the stable inverse `α=a+log1p(-exp(-a))`, no `expm1` overflow), so every fixed-T slice is provably convex in c — the property that motivates softplus over a polynomial amplitude (no spurious miscibility gaps). `slice_at(T)` returns a `_SoftplusSlice` with an analytic c-derivative (so the parent solver uses exact gradients); no terminal-phase requirement, unlike `CalphadSurface2DInterpolator`. Fit is multi-started (heuristic large-amplitude + fitted per-T `SoftplusFit` on the central slice) to dodge the flat near-zero-amplitude basin. Tests in `tests/unit/interpolate/test_softplus_surface.py` (in-family recovery, convexity under noise + extrapolation, analytic-derivative, shape contract, `Surface2DInterpolatingPhase` Gibbs–Duhem + convex `f(c)`).
- `whitney.py` — `WhitneyRBFInterpolator` (sklearn-style estimator; **internal-only**, not in `interpolate/__init__.py:__all__`) and `WhitneyTemperatureInterpolator` (the exported `Interpolator` strategy that wraps it). Used for Whitney-extension RBF interpolation that handles convex-hull projection.

**`landau/calculate.py`** — core thermodynamic calculations.
- `calc_phase_diagram(phases, Ts, mu=..., refine=True, ...)` — main entry point that builds the raw `(T, mu, phase, c, phi, stable, border, refined, locus, …)` dataframe. Every row carries a `locus` column (`landau.features.Locus`: `"interior"`/`"boundary"`/`"triple"`) classifying what part of the diagram it lives on (PR #235); grid samples and synthetic `±inf` frame edges stay `INTERIOR` even when `border=True`, refined rows get `BOUNDARY`/`TRIPLE` from `RefinedPoint.to_rows`. The per-phase `f_excess` endpoint reference compares candidate phases by the tangent-line value at the pure concentrations (`phi` at c=0, `phi + mu` at c=1) instead of `f` at the extreme sample (PR #231, follow-up to #155/#166) — `f(c)` is convex with slope `mu`, so a phase ending at `c = 1 - eps` lies below its own tangent value at `c = 1` by `~mu*eps` and would otherwise steal the reference from a line phase sitting exactly at `c = 1`, lifting that phase's `f_excess` off zero. Line phases located exactly at an endpoint are unchanged (both reduce to `f`). The per-temperature reduction is the module-level `_f_excess_tangent_chord(dd)` (promoted from an inline `sub` closure in PR #267) and is applied via `_apply_series` in `calc_phase_diagram`.
- `refine_phase_diagram(pdf, phases, min_c, max_c)` — orchestrator over a sequence of `Refiner` instances (defaults via `landau.refine.default_refiners`).
- `guess_mu_range(phases, T, samples, tolerance=1e-2)` — autodetect mu sampling window. Scans `c(mu)` on `(-10, 10)` with `scipy.optimize.brute(..., finish=None)` to locate the `c≈0`/`c≈1` ends, then `interp1d`-inverts the sampled `c(mu)` curve onto an even `c` grid (PR #275). `finish=None` is load-bearing: the default `fmin` polish is unconstrained and, because `c(mu)` saturates only asymptotically, walks far outside the bracket — that blows up the linspace spacing and oversamples the single-phase tails. The averaged-`c` reduction is shared with the regression test as `_semigrand_average_concentration(phases, T, mu)`.
- `cluster_T_c` / `cluster_T_c_mu` / `cluster` — agglomerative clustering for collapsing co-located transitions; `distance_threshold` is a **required** kwarg here. User-facing `cluster_phase` (`plot.py`) and `get_polygons` accept it with `default=0.5`.
- `get_transitions(df)` — extract phase-boundary rows.
- Private `_apply_series(grouped, fn, name)` (PR #242) wraps the pandas 2/3 single-group `groupby().apply()` workaround: runs `fn(g).to_frame(name)` per group and indexes back out, so a `Series`-returning callable composes consistently across pandas versions. Used by `calc_phase_diagram` (`f_excess`), `get_transitions` (`transition_unit`), and `plot.py::cluster_phase` (PR #254, closed #244 — replaced the inline copy).
- Private `_join_phase_unit` / `_split_phase_unit` shared with `poly.py` / `plot.py` (PR #132).

**`landau/refine.py`** — phase-boundary refinement strategies (PR #115 introduced the `Refiner` API).
- `Refiner` ABC: subclasses implement `propose(df, phases)` + `solve(candidate)` → `RefinedPoint` / `RefinedMiscibilityGap`. Base `run()` drops dominated/negative-T rows and packages output rows.
- Shipped refiners: `ScanRefiner` (1-D bisection between disagreeing samples; recurses via `_solve_pair` into both sub-brackets when a third phase dominates the located crossing, so a stable window narrower than the grid spacing still emits its two real transitions instead of leaving the candidate to be dropped by `_dominated` in `Refiner.run` — PR #232), `DelaunayLineRefiner` (one transition per two-phase simplex), `DelaunayTripleRefiner` (triple points from three-phase simplices; dedups via enclosing-simplex extents, PR #144), `ClausiusClapeyronRefiner` (predictor-corrector trace of a two-phase coexistence line, both T directions), `MiscibilityGapRefiner` (same idea for intra-phase miscibility splits). The last two share `_CCBase`.
- `default_refiners(df)` picks the set based on which of `mu`/`T` is sampled. For 2-D grids (both axes sampled) the default is `DelaunayTripleRefiner` + `ClausiusClapeyronRefiner` + `MiscibilityGapRefiner` — `DelaunayLineRefiner` and `ScanRefiner` are opt-in. For 1-D scans the default is just the matching `ScanRefiner`.
- A `boundary_id` int column tags refined rows by the coexistence line they belong to; `_CCBase.run()` assigns one id per `_pair_key`, base `Refiner.run()` assigns one id per non-empty solve call. Landed in PR #163, closes #124.
- `RefinedPoint.to_rows` stamps a `locus` of `Locus.TRIPLE` when three phases coexist and `Locus.BOUNDARY` otherwise; `RefinedMiscibilityGap.to_rows` always stamps `Locus.BOUNDARY` (PR #235). Both feed into `calc_phase_diagram`'s `locus` column.

**`landau/plot.py`** — phase-diagram plotting.
- `plot_phase_diagram` (c–T), `plot_mu_phase_diagram` (μ–T), `plot_1d_mu_phase_diagram`, `plot_1d_T_phase_diagram`. The mu/c variants share `_plot_phase_diagram` + the `_set_axis_for` helper (PR #122). Generalising the 1D/2D split is open (#34, #60). Both 2d variants take `triplepoints=False` (PR #247): when true `_plot_triplepoints` groups the refined `Locus.TRIPLE` rows by `(mu, T)` and draws one isothermal `hlines` per invariant in c-T (spanning the coexisting concentrations) or one black point marker in μ-T (where the three phases collapse). Unrefined frames carry no `TRIPLE` rows and draw nothing; a frame without a `locus` column is a no-op. The old `plot_phase_diagram(tielines=...)` keyword is kept as a `@deprecate(tielines=...)` alias that warns and maps onto `triplepoints`. 2d variants take `legend=True` (PR #248) plus `inline_legend=True` (PR #199): `legend=False` suppresses both labelling paths so the bare polygons render; `legend=True` (default) lets `inline_legend` choose between in-place labels (the legend box is dropped and each polygon is labelled at the pole of inaccessibility — `shapely.ops.polylabel`, coordinates pre-normalised by the axis data-ranges — with white-outlined black text via the shared `_text_with_outline` helper, which the 1d top-spine labels also use) and the seaborn legend box. `_add_inline_polygon_labels` falls back in two pixel-space steps when the horizontal label box does not fit its polygon (PR #228): rotate by 90° at the pole, then offset horizontally beside the polygon (left-flipped at the right axis edge, clamped into the axes both ways) so terminal line phases at c=0/c=1 stay visible. Off-polygon labels are no longer constrained to their own polygon, so a final pass via `_group_overlapping_intervals` + `_spread_labels` fans labels with overlapping horizontal extents apart vertically within the axes (PR #230). Both 1d functions assign per-segment `_seg_id`s via `_assign_segment_ids(df, scan_col)`: ordering by the cut axis (`'mu'` / `'T'`), a phase starts a new segment at each `stable` flip, so a phase unstable in two disjoint windows draws as separate dashed branches instead of one line through the stable region. Threshold-free and grid-density independent (PR #179, closed #177). `scan_col` is a parameter so it extends to generalised 1d cuts (#34, #60). Before drawing, `_bridge_unstable_segments(df, scan_col)` duplicates each refined transition (`border` + `stable`) row onto the adjacent unstable `_seg_id`s so a phase's dashed branch runs flush to the exact transition instead of starting at the next (possibly coarse) sample; the duplicate feeds `sns.lineplot` only, annotations still use the un-bridged frame. Both 1d functions also accept `reference_phase=None` (PR #189) — when set, subtracts that phase's `phi` along the cut via `_subtract_reference_phase` and updates the y-label. `_subtract_reference_phase` requires the reference phase to be sampled at every non-`border` grid point of the cut and raises `ValueError` pointing at `keep_unstable=True` otherwise (PR #273); without that guard `np.interp` silently clamps past the reference's stability window and chord-bridges interior gaps, corrupting every other phase's curve on a stable-only frame. Refined `border` rows are excluded from the check, so the standard `keep_unstable=True` flow (reference legitimately absent at a transition row, interpolated from the dense grid) is unchanged. The seaborn hue/style legend is replaced by `_add_1d_phase_legend` (top-spine ticks at transitions + right-end labels per phase, PR #190); colours are stashed on `ax._landau_phase_colors`. `top_labels`/`side_labels` flags toggle each element. When no sample lies strictly between two border rows (a stable window narrower than the grid, recovered by `ScanRefiner` per PR #232), the per-interval stable-phase lookup falls back to the phase marked stable on both enclosing rows; a stable-phase change with no border row at all still raises.

### Key design patterns

- **Frozen-dataclass + ABC** for `Phase` and concrete `Interpolator` subclasses (`PolyFit`, `SGTE`, `RedlichKister`, `StitchedFit`, `SoftplusFit`) — instances are immutable, structurally hashable. `Refiner` concrete subclasses are plain mutable classes (they accumulate state across `propose`/`solve` calls); `AbstractPolyMethod` is a non-frozen dataclass.
- **Semi-grand potential** is the central thermodynamic quantity; equilibrium = argmin over phases.
- **Interpolator strategy** plugged into `InterpolatingPhase` / `SlowInterpolatingPhase` decouples the analytical model (SGTE, RedlichKister, PolyFit, Softplus, Whitney) from the phase wrapper.
- **Refiner strategy** plugged into `refine_phase_diagram` decouples "where might a transition live?" from "find the exact location".

- Direct unit tests for the private `_resample_borders_once` helper landed in `tests/unit/test_resample.py` (PR #284): all returned rows carry `border=True` and finite `mu`, the `run` column propagates the input arg (default `0`), and nonzero noise shifts the refined A-B boundary away from the noiseless `mu=0` location on two terminal line phases.

Re-exported from `landau/__init__.py` and `landau/interpolate/__init__.py`. Tests in `tests/unit/interpolate/test_spline_fit.py` (strict interpolation, cubic round-trip, scalar/array shapes, piecewise-linear degree-1, degree clamping, smoothing denoising, scipy-default smoothing, frozen-dataclass immutability).
- `guess_mu_range` brute scan pinned to its `(-10, 10)` bracket via `finish=None` (PR #275). The default `fmin` polish is unconstrained and, because `c(mu)` saturates only asymptotically along the single-phase tails, walked the `mu0`/`mu1` endpoints out to ~-400, inflating the `linspace` spacing and oversampling the saturated regions (visible as the over-aggressive Au-Cu sampling). Round-trip test in `tests/unit/test_calculate.py` asserts that the averaged `c(mu_edge)` matches the reported `(c0, c1)` to `atol=1e-3` — fails on the old code (off by `tolerance`). The averaged concentration was extracted into `_semigrand_average_concentration` so the test inverts the same contract `guess_mu_range` does. The narrower sampling shifted the exact triple-point `mu`; `tests/unit/plot/test_triplepoint.py` was tightened to compare marker/line coordinates component-wise (the prior nested `pytest.approx((x, y))` applied no tolerance and matched only by a `mean()` bit-coincidence).
- `InterpolatingPhase` and `SlowInterpolatingPhase` public `semigrand_potential`/`concentration` interfaces gained unit tests (PR #265, closes the two remaining checkboxes in #22). 13 cases in `tests/unit/test_phases.py` covering scalar/array shape contracts, `[0, 1]` boundedness, `dmu`-monotonicity, Gibbs-Duhem `c = -dphi/dmu` (numerical gradient at `atol=5e-3`), and a `InterpolatingPhase` vs `SlowInterpolatingPhase` cross-check at the same tolerance. Shared `_INTERP_ATOL = 5e-3` constant.
- `check_interpolation` / `check_concentration_interpolation` (`landau/phases/__init__.py`) gained a `plot_error=False` flag (PR #272) — plots `line_free_energy(T) - sample` (temperature variant) or `free_energy(c) - line_value` (concentration variant; measured in entropy-adjusted coordinates so it is invariant to `plot_excess`). Tests in `tests/unit/test_phases.py` assert residuals match an independent recomputation to `atol=1e-9` under under-parametrised fits.

## Common Tasks

### Add a new phase type
1. Subclass `Phase` (or `AbstractLinePhase`) as a `@dataclass(frozen=True)` in `landau/phases/__init__.py` (or a new module under `landau/phases/`).
2. Implement `semigrand_potential(T, dmu)` and `concentration(T, dmu)`.
3. Add to `landau/phases/__init__.py:__all__` *and* re-export from `landau/__init__.py` if it's user-facing.
4. Tests: at minimum a `tests/unit/test_phases.py` case checking scalar/array input shapes plus a tight numerical assertion (loose `atol=0.05` "it ran" tests get rejected; see Working Style).

### Add an interpolation method
1. New class in `landau/interpolate/basic.py` (or a new module) inheriting `Interpolator` / `TemperatureInterpolator` / `ConcentrationInterpolator`.
2. Implement the interpolation interface.
3. Export from `landau/interpolate/__init__.py:__all__`.
4. Add Hypothesis round-trip tests (random coeffs in → `fit()` → coeffs out) under `tests/unit/interpolate/` (PR #82).

### Add a refiner
1. New `Refiner` subclass in `landau/refine.py` with `propose` + `solve`.
2. Add to `landau/refine.py:__all__`.
3. Decide whether it belongs in `default_refiners` (most don't; opt-in unless cheap).
4. Tests in `tests/unit/test_refine.py` against a small known-transition fixture; assert exact rows, not just non-empty.

### Render the visual-review plots
```bash
python tests/integration/testplots.py --only 2d_basics 2d_toy --poly-method fasttsp
```
Output: `tests/integration/_plots/*.png`. Inline-attach the PNG to a PR comment when diagnosing a plotting change.
